#### 训练参数

**Epoch**
走完全部数据一次 表示一个Epoch
`为什么要走那么多轮epoch而不是走一轮？`
因为模型刚开始的参数(weight,bias)全是随机的，模型仅仅走一轮数据，其预测出来的偏差(loss)是极高的，将导致欠拟合；
因此模型需要多走几轮epoch，更改参数，使之能够更好的拟合数据，预测
```
也可以理解为由于刚开始模型的参数全身随机的，而一个epoch只够模型对参数进行一个微小的变化，但是要最终匹配这个工作，参数要不断进行一次一次地微小修改，因此需要多走几轮epoch
```

**Batch_size**
表示每次训练时输入的样本数量
`小批量数据比单样本更加稳定 比全批量更加高效`


**Interation**
一次Interation表示完成一次Batch数据的前向传播和反向传播

#### SGD
**真正的SGD**
一次只使用一个样本 一个样本进行前向传播 一个样本进行反向传播 然后更新一次参数

**Mini-batch SGD**
一次使用一个小批次样本，一批一批的进行前向传播和反向传播 更新一次参数

#### One-hot编码和类别索引

**one-hot编码**
将类别映射为相互独立的二进制向量
如：\[0,0,0,1,0]

**类别索引**
是一个标量，这一个值就表示这条数据的类别
